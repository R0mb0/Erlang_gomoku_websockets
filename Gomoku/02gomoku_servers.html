Phase 3: Supervision & WebSockets

This document verifies the automatic startup of the Erlang Supervisor tree and tests the bidirectional JSON communication between the browser and the Cowboy WebSocket server.

Backend Erlang Console Tests

1

Verify Auto-Boot

Launch the shell. You should no longer need to call start_link() manually. Look for the ===> Booted gomoku_backend message.

rebar3 shell
2

Verify Supervisor Children

Check if the Supervisor automatically started our actors. The whereis function should return a PID (Process ID) instead of undefined.

whereis(gomoku_server).
3

Observe Traffic

Leave the terminal open. When you use the interactive tool on the right, you will see print statements appearing in the Erlang console confirming the reception of messages.
