Tuple Space Engine - Test Guide

Follow these steps in order to test the core mechanics of the distributed Tuple Space engine (Linda model). Copy the commands and paste them into your terminal.

1

Boot the Erlang Environment

Make sure you are in the gomoku_backend directory and start the Rebar3 shell. This will automatically compile your code.

rebar3 shell
2

Start the Server

Once inside the Erlang shell (1>), boot up the gen_server process. You should receive an {ok,<0.some_number>} response.

tuple_space:start_link().
3

Test Write Operation (OUT)

Let's simulate Player 1 placing a piece at coordinates X=5, Y=5. Since this is an asynchronous "cast", it will return ok immediately.

tuple_space:out({cell, 5, 5, player_1}).
4

Test Non-Destructive Read (RD)

Let's check if there's anything at 5,5. We use the special '_' atom (wildcard) to say "I don't care whose piece it is". The tuple will remain in memory.

tuple_space:rd({cell, 5, 5, '_'}).
5

Test Destructive Read (IN)

Now we physically extract the piece from the board. After this command, cell 5,5 will be empty again in our Tuple Space.

tuple_space:in({cell, 5, 5, '_'}).
⚠️

Test the Blocking Queue (The Real Magic!)

Now that we have removed the tuple with the previous in command, let's try to request it again. Paste this command:

tuple_space:in({cell, 5, 5, '_'}).

What just happened? Your shell froze! This is the exact expected behavior: your request has been parked in the server's waiting list because the tuple doesn't exist anymore.

How to unblock your terminal:

  1. Press Ctrl+G (the User switch command menu will appear).
  2. Press i (Interrupt) and hit Enter.
  3. Press c (Connect) and hit Enter to return to the normal shell.
